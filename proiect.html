Introduction

AICO is an application designed to offer solutions for airsoft players looking to enhance their experience and link up easier with people to play with. The application has 3 main components:

  • Organizing and finding games through the mobile application
  • Enhancing the experience on the field with AR technology
  • Referee interface for spectating the game to ensure a fair match
The interaction with the application is analyzed on 4 personas: The gamer, the athlete, the referee and the concerned parent.

Interaction types

Interaction types define the way the user sends and receives information to and from the application. The designed application includes a variety of interaction types and offers a fair chance to people who couldn't enjoy airsoft before due to certain disabilities.

  • Traditional: The referee is using a web interface to see real time footage from the field.
  • Touchscreen: The players are using the touchscreen interface of the mobile application in order to organize and find games.
  • Hand gestures: Players can use gestures to communicate different types of predefined callouts. The gestures are detected by the smartwatch and can be used for example for pings visible on the augmented field.
  • AR: The AR glasses provide the player a wide range of additional information such as the position of their teammates, objectives, spotted enemies, pings, gun ammo, zone restrictions and much more.
  • Voice: The players can communicate with eachother using the microphone in their smartwatch.
There are several helping features for people with disabilities. Deaf people can better visualize their surroundings and also receive help in the form of augmented visual markers that indicate where certain sounds are coming from. Mute people can communicate using the gesture system.

Story boards

Airsoft player experience

The main actor of the storyboard is Denis. He is using the AICO app to organize an airsoft match with his preffered rules. Other players join his game and meet with Denis at the decided location where they have a fun airsoft experience. They use the provided equipment including a smartwatch, a bluetooth connected weapon and some AR glasses. At the end of the game they can check their scores and stats during the match.

Referee experience

The main actor of the storyboard is Mihai. He decides to make some money by refereeing some airsoft matches as he is also passionate about this sport himself. He picks a game and on the selected date welcomes the players on the field. After making sure that everyone is present and understood the explained rules he monitors the players using the cameras integrated in the glasses to make sure everyone is playing fairly.

Sketches and Prototypes

Login Form Sketch

The Login page features fields for entering the e-mail and password of the user, whilst hiding the password initially to protect the user's data. There is a checkbox to remember the account on this device and a link to the register page in case the user does not have an account. They can also connect via popular social media platforms like facebook, google or twitter.

Register Page Sketch

The Register page features fields for entering the e-mail and password of the user, whilst hiding the password initially to protect the user's data. There is a checkbox to subscribe to the newsletter and a link to the login page in case the user already has an account.

Forgotten Password Page Sketch

The Recover Password page has two fields that serve as ways to recover the user's lost password. They can do so via e-mail or phone sms. There are also links to the register page and the login page.

Account Page Sketch

The Account Page stores all sort of information regarding the user's account, such as his personal information, payment methods and friends. On the top left the user's profile picture, name and number of games played are displayed, with a possibility to change the picture by clicking on it. In the personal information tab the user can see and change their e-mail or name associated with their account. The payment section allows them to see and edit card information. The frieds tab on the right incentivises the user to get a small community around them with whom to participate in airsoft matches.

Payment Page Sketch

The payment screen is developed for the mobile application only as organizing airsoft matches is done via the mobile application exclusively. It contains a recap of the information regarding the match, a section for choosing the payment method and the "pay" button. payment can be done either with a card, or in cash at location, the latter having added tax to incentivise users to pay online.

Game Browser Page Sketch

In the game browser page the referee can view listed games and their details, like the name of the host, player count, date and time, location and game mode. Once they find a suitable match and decide to referee for it they can press the corresponding button and lock it the position. They are then expected to show up at the match.

Cameras Page Sketch

In the cameras page the referee can watch the live feed of players in the arena. The camera layout can be adjusted using the menu on the right. There is a home button on the bottom, as well as an announcement button to make an announcement and a kick player button for dealing with people that do not follow the rules. In the left menu there are filters and sorting fields for finding certain players, as well as a list of players.

Cameras Page Prototype

Referee Overview Page Prototype

In this page the referee can see the name of the game, the current score, the time allocated for the match, the teams and the player's scores. They can also click the spectate button to be taked to the cameras page, or start the game when everyone is ready.

Mobile Dashboard Prototype

The Dashboard features an account button, a notifications section, a way to browse and organize games and a match history.

Browse Games Prototype

In this page the mobile user can see details of a match and ask for permission to join from the owner of the match. There are also filters in place for finding preffered games.

Organize Match Prototype

In this page the user can create a new match to be listed onto the server and give all sorts of details regarding it, like date, hour, gamemode, location, length and player count. After the settings are decided upon the user is taked to the payment page.

Your Games Prototype

In this page the user can see his current organized matched and see when they will start and who will attend.

History Prototype

In the match history part of the application the user can view their past matches and their results, as well as see their performance statistics during each match.

Notifications Prototype

In this screen the user is notified of game invitations, refereee announcements and any other messages that require his attention.

Settings Prototype

In this screen the user can change the language of the application and their password and logout of their account.
